# 디디딧 콘티 · Google 시트 연동

팀원은 **Google 스프레드시트**에서 콘티를 편집하고, 시나리오 머신은 **생성·동기화**만 담당합니다.

## 1. 스프레드시트 만들기

1. [Google 스프레드시트](https://sheets.google.com) 새로 만들기
2. 이름 예: `디디딧 콘티`
3. 팀원 이메일에 **편집자** 권한 공유

탭 구조 (Apps Script가 자동 생성해도 됨):

| 탭 이름 | 용도 |
|---------|------|
| `Xenics` | Xenics 프로젝트 콘티 |
| `콘티` | 기본/기타 프로젝트 |
| `_안내` | (선택) 사용 안내 |

1행 헤더 (고정):

```
대본 | 장면 | 사이즈 | 자막 | 코멘트
```

## 2. Apps Script 배포

1. 스프레드시트 → **확장 프로그램** → **Apps Script**
2. `Code.gs` 내용을 이 폴더의 `Code.gs`로 교체
3. **프로젝트 설정** → **스크립트 속성** 추가:

| 속성 | 값 |
|------|-----|
| `SPREADSHEET_ID` | URL의 `/d/` 와 `/edit` 사이 ID |
| `API_TOKEN` | 팀만 아는 임의 문자열 (예: `dddit-works-2026-xxxxx`) |

4. `setupDdditTemplate` 함수 선택 → **실행** (최초 1회, 탭 생성)
5. **배포** → **새 배포** → 유형: **웹 앱**
   - 실행 계정: **나**
   - 액세스: **모든 사용자**
6. 배포 URL 복사 (예: `https://script.google.com/macros/s/...../exec`)

## 3. 시나리오 머신 설정

1. https://works.mansejin.com/dddit/script/ 접속
2. 우측 상단 **API** → 하단 **Google 시트 연동**
3. 입력:
   - **시트 API URL**: 위 웹 앱 URL
   - **API 토큰**: 스크립트 속성과 동일
   - **시트 열기 URL**: 스프레드시트 주소 (팀 공유용)
4. 저장

## 4. 사용법

| 버튼 | 동작 |
|------|------|
| **시트로 보내기** | 콘티 표 전체를 시트 탭에 덮어쓰기 |
| **시트에서 불러오기** | 시트 내용을 머신으로 가져오기 (AI 수정 전) |
| **시트 열기** | 브라우저에서 스프레드시트 새 탭 |

프로젝트별 탭:

- `?project=xenics` → `Xenics` 탭
- 그 외 → `콘티` 탭

## 5. 팀 워크플로

1. 담당: 시나리오 머신에서 파트 생성 → **시트로 보내기**
2. 팀원: 시트에서 장면·자막 수정 (동시 편집 가능)
3. 담당: **시트에서 불러오기** → AI 수정 → 다시 **시트로 보내기**
4. 촬영팀: 시트 링크 북마크

## API 요약

| 메서드 | action | 설명 |
|--------|--------|------|
| GET | `meta` | 시트·탭 정보 |
| GET | `get&project=xenics` | 콘티 행 JSON |
| POST | `replace` | 전체 덮어쓰기 `{ token, project, rows }` |
| POST | `append` | 하단 추가 `{ token, project, rows }` |

모든 요청에 `token` 필요 (GET: 쿼리, POST: JSON body).

## 문제 해결

| 증상 | 확인 |
|------|------|
| Unauthorized | API_TOKEN 일치 여부 |
| SPREADSHEET_ID | 스크립트 속성 |
| CORS/네트워크 | 웹 앱 배포·액세스 "모든 사용자" |
| 빈 탭 | `setupDdditTemplate` 실행 또는 시트로 보내기 1회 |
